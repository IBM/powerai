    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-installing-ibm_db-and-ibm_db_dbi-module" class="anchor" aria-hidden="true" href="#installing-ibm_db-and-ibm_db_dbi-module"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installing ibm_db and ibm_db_dbi module</h1>
<p>We are assuming that you have Python already installed. In Linux you may need the python-dev package (you can install python-dev package through "$yum install python-devel" if yum doesn't work then you can also install it through "$apt-get install python-dev")</p>
<p>Note:The minimum python version supported by driver is python 2.7 and the latest version supported is python 3.7 except version 3.3 as it has reached end-of-life.</p>
<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation</h2>
<pre><code>  pip install ibm_db

  or

  easy_install ibm_db
</code></pre>
<p>This will install <em>ibm_db</em> and <em>ibm_db_dbi</em> module.</p>
<p>If you face problems due to missing python header files, you would need to install python developer package before installing python ibm_db driver.
e.g:</p>
<pre><code>    zypper install python-devel

directory. A valid config.py will need to be created to configure your DB2
settings. A config.py.sample exists that can be copied and modified for your
environment.</p>
<p>The config.py should look like this:</p>
<div class="highlight highlight-source-python"><pre><span class="pl-s1">test_dir</span> <span class="pl-c1">=</span>      <span class="pl-s">'tests'</span>         <span class="pl-c"># Location of testsuite file (relative to current directory)</span>

<span class="pl-s1">database</span> <span class="pl-c1">=</span>      <span class="pl-s">'test'</span>          <span class="pl-c"># Database to connect to</span>
<span class="pl-s1">user</span>     <span class="pl-c1">=</span>      <span class="pl-s">'db2inst1'</span>      <span class="pl-c"># User ID to connect with</span>
<span class="pl-s1">password</span> <span class="pl-c1">=</span>      <span class="pl-s">'password'</span>      <span class="pl-c"># Password for given User ID</span>
<span class="pl-s1">hostname</span> <span class="pl-c1">=</span>      <span class="pl-s">'localhost'</span>     <span class="pl-c"># Hostname</span>
<span class="pl-s1">port</span>     <span class="pl-c1">=</span>      <span class="pl-c1">50000</span>           <span class="pl-c"># Port Number</span></pre></div>
<p>Point the database to mydatabase as created by the following command.</p>
<p>The tests that ibm_db driver uses depends on a UTF-8 database.  This can be
created by running:</p>
<pre><code>  CREATE DATABASE mydatabase USING CODESET UTF-8 TERRITORY US
</code></pre>
<p>Some of the tests utilize XML functionality only available in version 9 or
later of DB2.  While DB2 v8.x is fully supported, two of the tests
(test_195.py and test_52949.py) utilize XML functionality.  These tests will
fail on version 8.x of DB2.</p>
<h2><a id="user-content-running-the-driver-testsuite-on-linux" class="anchor" aria-hidden="true" href="#running-the-driver-testsuite-on-linux"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Running the driver testsuite on Linux</h2>
<p>In order to run the entire python driver testsuite on Linux, run this
command at the command prompt:</p>
<pre><code>  python tests.py
</code></pre>
<p>To run a single test, set the environment variable, <strong>SINGLE_PYTHON_TEST</strong>, to
the test filename you would like to run, followed by the previous command.</p>
<h2><a id="user-content-running-the-driver-testsuite-on-windows" class="anchor" aria-hidden="true" href="#running-the-driver-testsuite-on-windows"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Running the driver testsuite on Windows</h2>
<p>In order to run the entire python driver testsuite on Windows, run this
command at the command prompt:</p>
<pre><code>  tests.py
</code></pre>
<p>To run a single test, set the environment variable, <strong>SINGLE_PYTHON_TEST</strong>, to
the test filename you would like to run, followed by the previous command.</p>

<p>If trusted context is not set up, there will be two failures related to trusted context. When thick client has been used than additioanl three failures related to create, recreate DB.</p>
<h2><a id="user-content-known-limitations-for-the-python-wrapper" class="anchor" aria-hidden="true" href="#known-limitations-for-the-python-wrapper"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Known Limitations for the Python wrapper</h2>

As such these might go unnoticed.</li>
